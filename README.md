# Маршруты — мобильный web-app для велопроката

Первая рабочая версия мобильного раздела/мини-приложения для готовых веломаршрутов по побережью.

## Что уже есть

- mobile-first интерфейс;
- главный экран с понятными кнопками;
- карточки маршрутов;
- один бесплатный маршрут;
- закрытые маршруты через код доступа;
- интерактивная карта Leaflet + OpenStreetMap;
- фильтры категорий точек;
- карточки точек с советами, туалетом, парковкой, перекусом и предупреждениями;
- кнопки открытия точки и маршрута в Яндекс.Картах;
- данные маршрутов в `data/routes.json`;
- простая статическая админка `admin.html` для проверки и скачивания обновлённого JSON.

## Как запустить локально

Из папки проекта:

```bash
python3 -m http.server 8080
```

Затем открыть:

```text
http://localhost:8080
```

Важно: не открывайте `index.html` напрямую через `file://`, потому что браузер может заблокировать загрузку `data/routes.json`.

## Код доступа для MVP

Коды лежат в `data/routes.json` → `meta.unlockCodes`.

Стартовые коды:

- `VELO2026`
- `KOSA`
- `ROUTE`

После ввода кода доступ сохраняется на телефоне в `localStorage`.

## Как заменить старт проката

В `data/routes.json` измените:

```json
"rental": {
  "name": "Старт проката",
  "address": "Ваш адрес",
  "lat": 54.9618,
  "lng": 20.4761
}
```

Также измените точку `rental-start` в массиве `points`, чтобы маркер старта совпадал с вашей реальной точкой.

## Как добавить точку

1. Добавьте объект в массив `points`.
2. Укажите уникальный `id`.
3. Выберите категорию из `categories`.
4. В нужном маршруте добавьте этот `id` в массив `points`.

Пример:

```json
{
  "id": "new-coffee",
  "category": "food",
  "title": "Кофе у моря",
  "description": "Партнёрская кофейня с водой и быстрым перекусом.",
  "lat": 54.96,
  "lng": 20.47,
  "fromRental": "8 мин / 1 км",
  "advice": "Лучше заезжать до дальнего маршрута.",
  "bikeParking": "Есть стойка у входа.",
  "toilet": "Уточнить у заведения.",
  "food": "Кофе, выпечка, вода.",
  "warnings": "Проверить сезонный график."
}
```

## Как подключить к действующему сайту

1. Загрузите папку проекта на хостинг или GitHub Pages.
2. На действующем сайте кнопку «Маршруты» ведите на URL этого раздела.
3. Для QR-кода используйте тот же URL.

## Что лучше сделать перед реальным запуском

- заменить координаты старта проката;
- проверить все координаты на месте;
- уточнить сезонность туалетов и кафе;
- добавить реальные партнёрские кафе;
- заменить демо-коды доступа;
- удалить кнопку «Демо: открыть для проверки» из `index.html`, если она не нужна;
- добавить ссылку на оплату или подключить YooKassa/CloudPayments/эквайринг.
